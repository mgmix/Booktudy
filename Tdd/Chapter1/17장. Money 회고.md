# 17장. Money 회고



### 테스트 프로세스

1. 작은 테스트를 추가한다
2. 모든 테스트를 실행하고, 실패하는 것을 확인한다
3. 코드에 변화를 준다
4. 모든 테스트를 실행하고, 성공하는 것을 확인한다
5. 중복을 제거하기 위해 리팩토링 한다



### 최종 검토

- 테스트를 확실히 돌아가게 만드는 세가지 접근법
  	- 가짜로 구현하기
  	- 삼각측량법
  	- 명백하게 구현하기
- 설계를 주도하기 위한 방법
  - 테스트 코드와 실제 코드 사이의 중복 제거
- 길이 미끄러우면 속도를 줄이고, 상황이 좋으면 속도를 높이는 테스트 사이의 간격을 조절할 수 있는 능력
